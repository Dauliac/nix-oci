nixosConfig.mainService Link copied!

Type: null or string

The NixOS service that this container runs.

When set, the container's package is auto-derived from the service module's package option, and an entrypoint wrapper script is generated from the service's systemd unit (preStart, ExecStartPre, ExecStart, directory creation).

Cannot be set together with package - they are mutually exclusive sources for the container's main program.

When null, nixosConfig only provides config files, users/groups, and extra packages (environment.systemPackages) as dependencies.

Default: null
Example: "nginx"
Declared in: nix/modules/oci/containers/nixosConfig/mainService.nix

nixosConfig.modules Link copied!

Type: list of unspecified value

NixOS modules to evaluate for this container.

These modules are evaluated with boot.isContainer = true and the resulting config files, packages, users/groups, and entrypoint are extracted into the container image.

Only one service should be enabled per container (no init system). Use nixosConfig.mainService to designate which service this container runs.

Default: [ ]
Example: 
[
  ({ pkgs, ... }: {
    services.nginx = {
      enable = true;
      virtualHosts."app".root = "/var/www";
    };
  })
]
Declared in: nix/modules/oci/containers/nixosConfig/modules.nix
